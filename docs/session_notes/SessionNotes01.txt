3) PHASE 2 — ERROR HANDLING (COMPLETE)
=====================================================
Implemented:
A) Error Response Contract
- ApiErrorResponse (single consistent error DTO):
  - timestamp (Instant / UTC)
  - status (int)
  - code (ErrorCode enum)
  - message (safe human-readable)
  - path (request URI)
  - (optional) details reserved for future (not required for MVP)

B) Central Exception Handling
- Global @RestControllerAdvice:
  - MethodArgumentNotValidException -> 400 VALIDATION_FAILED
  - DomainException -> ex.status + ex.code
  - AccessDeniedException -> 403 FORBIDDEN (controller-layer cases)
  - Exception (catch-all) -> 500 INTERNAL_ERROR (safe response)

C) Domain Exceptions
- DomainException base carries:
  - ErrorCode code
  - HttpStatus status
  - message
- Specific exceptions created:
  - ImmutableRecordException
  - InvalidTrainingException
  - EntityNotFoundException
  - InactiveEntityException

D) Security (401/403)
- Custom handlers added so security failures return same ApiErrorResponse JSON:
  - RestAuthenticationEntryPoint -> 401 UNAUTHENTICATED
  - RestAccessDeniedHandler -> 403 FORBIDDEN

E) Logging Rules
- Client never sees stack traces.
- Server logs full exception details (warn/error with exception object).
- User/entity IDs included via:
  - CurrentUser helper (SecurityContextHolder) for username
  - entity IDs in exception messages and/or log statements

4) SECURITY CONFIG — JSON 401/403 FOR /api/** (COMPLETE)
=====================================================
Why:
- Security errors occur in filter chain BEFORE controllers -> @ControllerAdvice won’t catch.
- Need JSON responses for /api/** while keeping browser redirect behaviour for UI pages.

Matcher change due to Spring Security version:
- AntPathRequestMatcher was not usable (deprecated/removed depending on version).
- Replaced with PathPatternRequestMatcher.

Implementation:
- Define matcher:
  var apiMatcher = PathPatternRequestMatcher.pathPattern("/api/**");

- Configure HttpSecurity.exceptionHandling:
  - defaultAuthenticationEntryPointFor(new RestAuthenticationEntryPoint(mapper), apiMatcher)
  - defaultAccessDeniedHandlerFor(new RestAccessDeniedHandler(mapper), apiMatcher)

Form login:
- Keep formLogin for UI, defaultSuccessUrl("/home", true)

CSRF:
- Disabled for MVP (csrf.disable()) — revisit later once browser POST forms exist.

5) JACKSON 3 / SPRING BOOT 4 MIGRATION ISSUE (RESOLVED)
=====================================================
Observed:
- DI mismatch / “ObjectMapper is red”
- Type mismatch:
  tools.jackson.databind.json.JsonMapper cannot be converted to com.fasterxml.jackson.databind.ObjectMapper

Root cause:
- Spring Boot 4 defaults to Jackson 3:
  - core mapper types live in tools.jackson.*
- Must not mix Jackson 2 (com.fasterxml...) mapper types with Jackson 3 (tools.jackson...) mapper types.

Fix rule (VERY IMPORTANT):
- Use ONE mapper type consistently across project:
  - tools.jackson.databind.json.JsonMapper (preferred)
- Remove any usage of:
  com.fasterxml.jackson.databind.ObjectMapper

Actions taken:
- Updated SecurityConfig + security handlers to use JsonMapper consistently.
- Kept jackson annotations imports where applicable (annotations may still be com.fasterxml.jackson.annotation.*)

6) LOGGING / MDC (ADDED & VERIFIED)
=====================================================
Goal:
- Include request correlation metadata in every log line:
  - requestId
  - user
  - path

Implementation:
A) MDC Filter
- Added LoggingContextFilter extends OncePerRequestFilter:
  - Set MDC keys at start of request:
    MDC.put("requestId", UUID or X-Request-Id header)
    MDC.put("user", CurrentUser.usernameOrSystem())
    MDC.put("path", request.getRequestURI())
  - Clear MDC in finally:
    MDC.clear()

B) CurrentUser helper
- Utility method returns username, or "SYSTEM" if not authenticated/available.
- Note: “anonymousUser” can exist as a technical placeholder in SecurityContext; we normalize to SYSTEM if needed.

C) Logback config
- Added src/main/resources/logback-spring.xml to print MDC values:
  - pattern includes:
    [req=%X{requestId}] [user=%X{user}] [path=%X{path}]
- Pitfall fixed:
  - Logback failed to start due to extra blank line BEFORE <?xml ...?>.
  - XML declaration must be first character in the file.
